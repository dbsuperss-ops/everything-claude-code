# 검증 명령 (Verify Command)

구현 내용을 확인하기 위해 종합 검증 루프를 실행합니다: $ARGUMENTS

## 임무

다음의 포괄적인 검증을 수행하십시오:

1. **타입 체크**: `npx tsc --noEmit`
2. **린트(Lint)**: `npm run lint`
3. **단위 테스트**: `npm test`
4. **통합 테스트**: `npm run test:integration` (가능한 경우)
5. **빌드**: `npm run build`
6. **커버리지 확인**: 80% 이상의 커버리지 확인

## 검증 체크리스트

### 코드 품질
- [ ] TypeScript 에러 없음
- [ ] 린트 경고 없음
- [ ] console.log 구문 없음
- [ ] 함수 길이 < 50줄
- [ ] 파일 길이 < 800줄

### 테스트
- [ ] 모든 테스트 통과
- [ ] 커버리지 >= 80%
- [ ] 엣지 케이스 포함 여부
- [ ] 에러 조건 테스트 여부

### 보안
- [ ] 하드코딩된 비밀 정보 없음
- [ ] 입력값 검증 포함 여부
- [ ] SQL 인젝션 리스크 없음
- [ ] XSS 취약점 없음

### 빌드
- [ ] 빌드 성공
- [ ] 경고 메시지 없음
- [ ] 번들 크기 적정성

## 검증 보고서 형식

### 요약
- 상태: ✅ 통과 (PASS) / ❌ 실패 (FAIL)
- 점수: 전체 Y개 중 X개 통과

### 상세 내용
| 점검 항목 | 상태 | 비고 |
|-------|--------|-------|
| TypeScript | ✅/❌ | [상세 내용] |
| Lint | ✅/❌ | [상세 내용] |
| Tests | ✅/❌ | [상세 내용] |
| Coverage | ✅/❌ | XX% (목표: 80%) |
| Build | ✅/❌ | [상세 내용] |

### 조치 사항
[실패(FAIL)인 경우, 수정해야 할 항목 목록]
