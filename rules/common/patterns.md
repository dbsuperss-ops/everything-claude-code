# 공통 패턴 (Common Patterns)

## 스켈레톤 프로젝트 (Skeleton Projects)

새로운 기능을 구현할 때:
1. 잘 검증된 스켈레톤 프로젝트를 검색하십시오.
2. 병렬 에이전트를 사용하여 옵션을 평가하십시오:
   - 보안 평가
   - 확장성 분석
   - 관련성 점수 산정
   - 구현 계획 수립
3. 가장 적합한 것을 기반으로 클론(Clone)하십시오.
4. 검증된 구조 내에서 반복 작업을 수행하십시오.

## 디자인 패턴

### 리포지토리 패턴 (Repository Pattern)

일관된 인터페이스 뒤에 데이터 접근을 캡슐화하십시오:
- 표준 작업 정의: findAll, findById, create, update, delete
- 구체적인 구현체는 저장소 세부 사항(데이터베이스, API, 파일 등)을 처리함
- 비즈니스 로직은 저장 메커니즘이 아닌 추상 인터페이스에 의존함
- 데이터 소스를 쉽게 교체할 수 있으며 목(Mock)을 이용한 테스트를 단순화함

### API 응답 형식

모든 API 응답에 일관된 봉투(Envelope) 형식을 사용하십시오:
- 성공/상태 표시기를 포함하십시오.
- 데이터 페이로드(성공 시 데이터, 에러 시 null)를 포함하십시오.
- 에러 메시지 필드(에러 시 메시지, 성공 시 null)를 포함하십시오.
- 페이지네이션 응답을 위한 메타데이터(total, page, limit)를 포함하십시오.
