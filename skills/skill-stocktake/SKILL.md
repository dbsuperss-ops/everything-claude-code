# 스킬 재고 조사 (skill-stocktake)

모든 Claude 스킬과 커맨드를 품질 체크리스트와 AI의 종합적 판단에 따라 감사하는 슬래시 커맨드(`/skill-stocktake`)입니다. 최근 변경된 스킬만 점검하는 **Quick Scan**과 전체를 리뷰하는 **Full Stocktake** 모드를 지원합니다.

## 대상 범위

커맨드가 실행된 위치를 기준으로 다음 경로를 탐색합니다:
- `~/.claude/skills/`: 모든 프로젝트에 적용되는 글로벌 스킬
- `{cwd}/.claude/skills/`: 현재 위치한 프로젝트 전용 스킬 (존재하는 경우)

**Phase 1 시작 시, 어떤 경로가 발견되고 스캔되었는지 명확히 표시합니다.**

## 실행 모드

- **Quick Scan**: 지난 실행 이후 변경된 스킬만 재평가합니다. (약 5~10분 소요)
- **Full Stocktake**: 전체 스킬을 처음부터 리뷰합니다. (약 20~30분 소요)
- 결과는 `~/.claude/skills/skill-stocktake/results.json`에 캐싱됩니다.

## 평가 프로세스 (Phase 2)

서브 에이전트(Opus 모델 권장)가 각 스킬을 읽고 다음 체크리스트를 적용합니다:
- [ ] 다른 스킬과의 콘텐츠 중복 여부 확인
- [ ] MEMORY.md / CLAUDE.md와의 중복 여부 확인
- [ ] 기술적 참조의 최신성 확인 (CLI 플래그, API, 도구 이름 등 웹 검색 활용)
- [ ] 사용 빈도 고려

### 평가 판정 (Verdict)
- **Keep**: 유용하고 최신 상태임.
- **Improve**: 유지할 가치가 있으나 특정 부분의 개선이 필요함.
- **Update**: 참조하는 기술이 구식임 (웹 검색으로 검증).
- **Retire**: 품질이 낮고, 너무 오래되었으며, 비용 대비 효용이 낮음.
- **Merge into [X]**: 다른 스킬과 상당 부분이 겹침. 통합 대상을 명시하십시오.

## 종합적 AI 판단 기준
- **실행 가능성 (Actionability)**: 즉시 따라 할 수 있는 코드 예제나 단계가 포함되어 있는가?
- **범위 적절성 (Scope fit)**: 이름과 내용이 일치하며 너무 넓거나 좁지 않은가?
- **고유성 (Uniqueness)**: MEMORY.md나 다른 스킬로 대체 불가능한 고유한 가치가 있는가?
- **최신성 (Currency)**: 현재 환경에서 기술 참조가 유효하게 작동하는가?

## 정리 및 통합 (Phase 4)
리뷰 결과에 따라 사용자에게 다음과 같이 제안합니다:
- **Retire / Merge**: 왜 제거하거나 통합해야 하는지 구체적인 이유와 대체 수단을 제시하고 사용자의 최종 승인을 받습니다.
- **Improve**: 무엇을 왜 바꿔야 하는지 설명하고 개선 여부를 사용자가 결정하게 합니다.
- **Update**: 최신 정보로 업데이트된 콘텐츠를 제시합니다.

**기억하십시오**: 스킬 재고 조사는 쓸모없는 정보(Clutter)를 줄이고, 에이전트가 항상 가장 정확하고 효율적인 가이드라인을 따르도록 유지하는 중요한 도구입니다.
