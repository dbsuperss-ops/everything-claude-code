# Go 개발 패턴 (Go Development Patterns)

견고하고 효율적이며 유지보수가 쉬운 애플리케이션을 구축하기 위한 관용적인(Idiomatic) Go 패턴과 최선 관행을 안내합니다.

## 활성화 시점

- 새로운 Go 코드를 작성할 때
- Go 코드를 검토(Review)할 때
- 기존 Go 코드를 리팩토링할 때
- Go 패키지 및 모듈을 설계할 때

## 핵심 원칙

### 1. 단순함과 명확성 (Simplicity and Clarity)
Go는 기교(Cleverness)보다 단순함을 선호합니다. 코드는 읽기 쉽고 의도가 명확해야 합니다. "행복한 경로(Happy path)"를 들여쓰기 없이 유지하고 에러 처리를 먼저 하십시오.

### 2. 제로 값(Zero Value)을 유용하게 만들기
별도의 초기화 없이도 기본값(Zero value) 상태에서 바로 사용할 수 있도록 타입을 설계하십시오. (예: `sync.Mutex`, `bytes.Buffer`)

### 3. 인터페이스를 받고, 구조체를 반환하기 (Accept Interfaces, Return Structs)
함수는 인터페이스를 파라미터로 받고, 구체적인 타입(Concrete type)을 반환하는 것이 기본입니다.

## 에러 처리 패턴

- **에러 래핑(Wrapping)**: `fmt.Errorf("...: %w", err)`를 사용하여 맥락을 추가하십시오.
- **Sentinel 에러**: `ErrNotFound`와 같이 공통적인 에러를 변수로 정의하십시오.
- **에러 검사**: `errors.Is`와 `errors.As`를 사용하여 래핑된 에러를 안전하게 확인하십시오.
- **절대 에러를 무시하지 마십시오**: 빈 식별자(`_`)로 에러를 넘기지 말고 반드시 처리하거나 기록하십시오.

## 동시성 패턴 (Concurrency Patterns)

- **Worker Pool**: 고루틴(Goroutine)과 채널(Channel)을 사용하여 병렬 작업을 제어합니다.
- **Context 활용**: `context.Context`를 통해 타임아웃, 취소 신호를 전파하십시오.
- **Graceful Shutdown**: OS 신호를 감지하여 서버를 안전하게 종료하는 로직을 구현하십시오.
- **고루틴 누수 방지**: 컨텍스트 취소 시 채널 송신이 블록되지 않도록 버퍼드 채널이나 `select` 문을 사용하십시오.

## 인터페이스 설계

- **작고 집중된 인터페이스**: `io.Reader`, `io.Writer`처럼 하나의 메서드만 가진 인터페이스를 선호하십시오.
- **사용하는 곳에서 인터페이스 정의**: 제공하는 패키지가 아닌, 기능을 사용하는 소비자(Consumer) 패키지에서 인터페이스를 정의하십시오.

## 메모리 및 성능

- **슬라이스 미리 할당**: 크기를 알 수 있다면 `make([]T, 0, capacity)`로 메모리 재할당을 방지하십시오.
- **sync.Pool**: 빈번하게 생성되고 파괴되는 객체(예: 버퍼)에 대해 객체 풀을 사용하여 GC 부하를 줄이십시오.
- **strings.Builder**: 루프 내에서의 문자열 결합은 `+` 대신 `strings.Builder`를 사용하십시오.

## 안티 패턴 주의

- 긴 함수에서 이름 없는 반환(Naked returns)을 사용하지 마십시오.
- 제어 흐름을 위해 `panic`을 사용하지 마십시오. 오직 치명적인 에러에만 사용하십시오.
- 구조체 내부에 `context.Context`를 저장하지 마십시오. 무조건 함수의 첫 번째 인자로 전달하십시오.
- 값 리시버와 포인터 리시버를 섞어서 사용하지 마십시오. 하나를 정해서 일관성을 유지하십시오.

**기억하십시오**: Go 코드는 "지루할 정도로" 예측 가능하고 일관성이 있어야 합니다. 기교를 부리기보다 명확하고 단순한 코드를 지향하십시오.
