# 콘텐츠 엔진 (Content Engine)

하나의 아이디어를 모든 곳에 똑같이 올리는 대신, 각 플랫폼에 최적화된 강력한 콘텐츠로 변환합니다.

## 활성화 시점

- X 포스트나 스레드 작성 시
- LinkedIn 포스트나 출시 업데이트 초안 작성 시
- 숏폼 영상이나 YouTube 설명 영상 스크립트 작성 시
- 기사, 팟캐스트, 데모 또는 문서를 소셜 콘텐츠로 재가공(Repurposing)할 때
- 출시, 마일스톤 또는 특정 주제를 중심으로 가벼운 콘텐츠 계획을 세울 때

## 초기 확인 사항

다음을 명확히 하십시오:
- 소스 자산: 무엇을 기반으로 가공하는가
- 대상 독자: 개발자, 투자자, 고객, 운영자 또는 일반 대중
- 플랫폼: X, LinkedIn, TikTok, YouTube, 뉴스레터 또는 다중 플랫폼
- 목표: 인지도 제고, 전환, 채용, 권위 확보, 출시 지원 또는 참여 유도

## 핵심 규칙

1. 플랫폼에 맞게 조정하십시오. 동일한 문구를 여러 곳에 그대로 올리지 마십시오.
2. 요약보다 후킹(Hook)이 더 중요합니다.
3. 각 포스트는 하나의 명확한 아이디어를 담아야 합니다.
4. 슬로건보다는 구체적인 내용을 사용하십시오.
5. 요청 사항(CTA)은 작고 명확하게 유지하십시오.

## 플랫폼별 가이드

### X (트위터)
- 빠르게 시작하십시오.
- 포스트당 또는 스레드의 트윗당 하나의 아이디어를 담으십시오.
- 꼭 필요한 경우가 아니면 본문에 링크를 넣지 마십시오.
- 해시태그 남발을 피하십시오.

### LinkedIn
- 강렬한 첫 줄을 사용하십시오.
- 문단을 짧게 유지하십시오.
- 교훈, 결과 및 시사점(takeaways)을 더 명확하게 드러내십시오.

### TikTok / 숏폼 영상
- 처음 3초 안에 시선을 사로잡아야 합니다.
- 단순 나레이션이 아닌 시각적 요소 중심으로 스크립트를 짜십시오.
- 하나의 데모, 하나의 주장, 하나의 CTA 구조를 유지하십시오.

### YouTube
- 결과를 초반에 보여주십시오.
- 챕터별로 구조화하십시오.
- 20~30초마다 시각적인 변화를 주십시오.

### 뉴스레터
- 관련 없는 항목들의 묶음이 아닌, 하나의 명확한 관점을 전달하십시오.
- 섹션 제목을 훑어보기 쉽게 만드십시오.
- 도입부 문단이 실질적인 역할을 하도록 구성하십시오.

## 재가공(Repurposing) 흐름

기본 단계:
1. 앵커 자산(Anchor asset) 선정: 기사, 영상, 데모, 메모 또는 출시 문서
2. 3~7개의 핵심 아이디어 추출
3. 플랫폼별 최적화된 변형 문구 작성
4. 출력물 간의 중복 내용 제거
5. 플랫폼의 성격에 맞게 CTA 조정

## 최종 결과물

캠페인 요청 시 다음을 반환하십시오:
- 핵심 관점 (Core angle)
- 플랫폼별 초안
- (선택 사항) 게시 순서
- (선택 사항) CTA 변형안
- 발행 전 필요한 추가 정보 목록

## 품질 게이트 (Quality Gate)

최종 결과물 전달 전 확인 사항:
- 각 초안이 해당 플랫폼의 분위기에 어울리는가
- 후킹(Hook) 문구가 강렬하고 구체적인가
- 일반적인 과장된 표현이 없는가
- 요청하지 않은 한 플랫폼 간 문구가 중복되지 않는가
- CTA가 콘텐츠 및 독자와 일치하는가
