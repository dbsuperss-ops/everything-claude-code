# Python 개발 패턴 (Python Development Patterns)

견고하고 효율적이며 유지보수가 쉬운 애플리케이션을 구축하기 위한 관용적인(Pythonic) 패턴과 최선 관행을 안내합니다.

## 활성화 시점

- 새로운 Python 코드를 작성할 때
- Python 코드를 검토(Review)할 때
- 기존 Python 코드를 리팩토링할 때
- 패키지 및 모듈을 설계할 때

## 핵심 원칙

### 1. 가독성 (Readability Counts)
명확하고 이해하기 쉬운 코드가 최우선입니다. 기교를 부리기보다 직관적인 코드를 작성하십시오.

### 2. 명시적인 것이 암시적인 것보다 낫다 (Explicit is Better Than Implicit)
마법 같은 기능(Magic)을 피하고, 코드가 하는 일을 명확하게 드러내십시오.

### 3. EAFP 방식 선호
"허락보다 용서를 구하는 것이 쉽다(Easier to Ask Forgiveness Than Permission)"는 원칙에 따라, 사전 조건 체크보다는 예외 처리(`try-except`)를 활용하십시오.

## 타입 힌트 (Type Hints)

- **기본 타입 어노테이션**: 인자와 반환값에 타입을 명시하십시오.
- **현대적인 타입 힌트 (3.9+)**: `List`, `Dict` 대신 `list`, `dict`와 같은 내장 타입을 사용하십시오.
- **Protocol**: 덕 타이핑(Duck typing)을 정적으로 검사하기 위해 `Protocol`을 사용하십시오.

## 에러 처리 패턴

- **특정 예외 잡기**: 구체적인 예외 상황을 캐치하고, `raise ... from e`를 통해 원인 예외를 보존하십시오.
- **커스텀 예외 계층**: 애플리케이션 전용 기본 예외 클래스를 만들어 관리하십시오.

## 컨텍스트 매니저 (Context Managers)

- 리소스 관리(파일, DB 연결 등)에는 항상 `with` 문을 사용하십시오.
- `contextlib.contextmanager`를 사용하여 간단한 커스텀 컨텍스트 매니저를 만드십시오.

## 컴프리헨션 및 제너레이터

- **리스트 컴프리헨션**: 단순한 변환 작업에 사용하되, 너무 복잡해지면 일반 루프로 풀어서 작성하십시오.
- **제너레이터**: 대량의 데이터를 메모리 효율적으로 처리할 때 `yield`를 사용하십시오.

## 데이터 클래스 (Data Classes)

- 단순 데이터 컨테이너에는 `@dataclass`를 사용하십시오. 자동 생성되는 메서드와 포스트 초기화 검증(`__post_init__`)을 활용할 수 있습니다.

## 안티 패턴 주의

- **가변 기본 인자**: `items=[]`와 같은 기본값 사용을 피하고 `None`으로 초기화하십시오.
- **타입 비교**: `type()` 대신 `isinstance()`를 사용하십시오.
- **None 비교**: `== None` 대신 `is None`을 사용하십시오.
- **와일드카드 임포트**: `from module import *`를 절대 사용하지 마십시오.

**기억하십시오**: Python 코드는 읽기 쉽고, 명시적이어야 하며, 놀라움을 최소화해야 합니다. 기교보다 명확성을 우선하십시오.
