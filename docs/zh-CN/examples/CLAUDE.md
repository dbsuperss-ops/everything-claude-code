# 프로젝트 가이드 (CLAUDE.md 예시)

이 파일은 Claude Code가 이 프로젝트의 코드를 이해하고 작업할 때 준수해야 할 핵심 지침을 담고 있습니다. 프로젝트 루트 디렉토리에 배치하십시오.

## 프로젝트 개요

[여기에 프로젝트의 목적, 주요 기능 및 기술 스택을 간략히 설명하십시오.]

## 핵심 규칙

### 1. 코드 구조 및 조직
* **모듈화**: 소수의 거대 파일보다 다수의 작은 파일을 선호합니다.
* **응집도와 결합도**: 높은 응집도와 낮은 결합도를 유지하십시오.
* **파일 크기**: 파일당 약 200-400줄, 최대 800줄 이내로 유지하십시오.
* **도메인 중심**: 기술 타입(Type)보다 기능/도메인별로 조직하십시오.

### 2. 코드 스타일 및 관행
* **이모지 금지**: 코드, 주석, 문서에 이모지를 사용하지 마십시오.
* **불변성(Immutability)**: 객체나 배열을 직접 수정하지 말고 항상 새로운 복사본을 생성하십시오.
* **로그 관리**: 프로덕션 코드에 `console.log`를 남기지 마십시오.
* **에러 처리**: `try/catch`를 통해 예외 상황을 명확히 처리하십시오.
* **입력 검증**: Zod 등을 활용하여 외부 입력값을 엄격히 검증하십시오.

### 3. 테스트 및 품질 보장
* **TDD 실천**: 기능 구현 전 테스트 코드를 먼저 작성하십시오.
* **커버리지**: 최소 80% 이상의 테스트 커버리지를 보장하십시오.
* **테스트 유형**: 유틸리티는 유닛 테스트, API는 통합 테스트, 핵심 여정은 E2E 테스트를 수행하십시오.

### 4. 보안 가이드라인
* **비밀값 보호**: API 키나 토큰을 코드에 하드코딩하지 마십시오. (환경 변수 사용)
* **쿼리 안전**: 반드시 파라미터화된 쿼리를 사용하여 인젝션을 방지하십시오.
* **CSRF 보호**: 상태 변경이 발생하는 엔드포인트에 보안 처리를 활성화하십시오.

## 전형적인 폴더 구조

```text
src/
├── app/              # 프레임워크 라우터 및 페이지
├── components/       # 재사용 가능한 UI 컴포넌트
├── hooks/            # 커스텀 훅 및 로직
├── lib/              # 유틸리티 및 라이브러리 설정
├── types/            # TypeScript 타입 정의
```

## 주요 명령어 (Slash Commands)

* `/tdd`: 테스트 주도 개발 워크플로우 실행
* `/plan`: 구현 계획 수립 및 검토
* `/code-review`: 코드 품질 및 보안 리뷰 수행
* `/build-fix`: 빌드 에러 자동 수정 시도

## Git 워크플로우

* **커밋 메시지**: `feat:`, `fix:`, `refactor:`, `docs:`, `test:` 접두사를 사용하십시오.
* **안전한 머지**: 메인 브랜치 직접 커밋을 지양하고 PR 리뷰를 거치십시오. 모든 테스트 통과가 머지의 필수 조건입니다.
