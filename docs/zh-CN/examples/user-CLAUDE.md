# 사용자 레벨 가이드 (USER CLAUDE.md 예시)

이 파일은 모든 프로젝트에 걸쳐 전역적으로 적용되는 Claude Code의 개인적인 코딩 습관과 선호도를 담고 있습니다. `~/.claude/CLAUDE.md` 경로에 배치하십시오.

## 핵심 철학

당신은 Claude Code입니다. 복잡한 작업이나 고도의 전문성이 필요한 영역에는 전용 에이전트와 도구를 활용하십시오.

**핵심 원칙:**
1. **에이전트 우선**: 복잡한 작업은 전문 에이전트에게 위임하십시오.
2. **병렬 실행**: 가능한 경우 여러 에이전트를 동시에 활용하여 작업 효율을 높이십시오.
3. **선 설계 후 구현**: 복잡한 작업 시작 전에는 반드시 구현 계획을 수립하십시오.
4. **테스트 주도**: 실제 코드 작성 전 테스트 코드를 먼저 작성하십시오.
5. **보안 지상주의**: 어떤 상황에서도 보안 가이드라인을 타협하지 마십시오.

상세 가이드는 `~/.claude/rules/` 폴더에 분류하여 관리하십시오:

| agents.md | 에이전트 오케스트레이션 및 상황별 선택 기준 |

## 개인 선호도 (Personal Preferences)

### 1. 보안 및 개인정보 보호
* **로그 관리**: API 키, 토큰, 비밀번호 등은 절대 로그에 남기거나 노출하지 마십시오.
* **출력 검증**: 외부 공유 전 민감 데이터 포함 여부를 반드시 재검토하십시오.

### 2. 코드 스타일
* **이모지 금지**: 코드, 주석, 문서 작업 시 이모지를 사용하지 마십시오.
* **불변성(Immutability)**: 객체나 배열을 직접 수정하는 대신 스프레드 연산자 등을 활용하십시오.
* **파일 분할**: 소수의 큰 파일보다 다수의 작은 파일을 선호합니다. (파일당 최대 800줄 제한)

### 3. Git 및 품질
* **커밋 원칙**: `feat:`, `fix:`, `refactor:`, `docs:`, `test:` 접두사를 활용한 원자적 커밋을 유지하십시오.
* **테스트 준수**: 배포 전 로컬 테스트 통과는 필수이며, 최소 80% 이상의 커버리지를 지향하십시오.

**철학 요약**: 에이전트 중심의 협업, 병렬 처리의 극대화, 구현 전 철저한 계획 및 테스트, 그리고 최우선 순위의 보안.
