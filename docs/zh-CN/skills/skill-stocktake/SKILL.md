# skill-stocktake

모든 Claude 스킬과 명령의 품질을 체크리스트 및 AI 종합 판단으로 감사하는 슬래시 명령(`/skill-stocktake`)입니다. 두 가지 모드를 지원합니다: 최근 변경 사항만 확인하는 **퀵 스캔(Quick Scan)**과 전체 리뷰를 위한 **풀 스톡테이크(Full Stocktake)**.

## 감사 범위

명령이 실행되는 디렉터리를 기준으로 다음 경로를 탐색합니다:

| 경로 | 설명 |
|------|-------------|
| `~/.claude/skills/` | 글로벌 스킬 (모든 프로젝트 공통) |
| `{cwd}/.claude/skills/` | 프로젝트 전용 스킬 (해당 디렉터리가 존재할 경우) |

**1단계 시작 시 발견된 경로와 스캔 범위를 명확히 표시합니다.**

### 특정 프로젝트 대상 실행

프로젝트 전용 스킬을 포함하려면 해당 프로젝트의 루트 디렉터리에서 명령을 실행하십시오:

```bash
cd ~/my-project
/skill-stocktake
```

## 실행 모드

| 모드 | 트리거 조건 | 소요 시간 |
|------|---------|---------|
| **퀵 스캔** | 기존 `results.json`이 존재할 때 (기본값) | 5~10분 |
| **풀 스톡테이크** | `results.json`이 없거나 `/skill-stocktake full` 입력 시 | 20~30분 |

**결과 저장 경로**: `~/.claude/skills/skill-stocktake/results.json`

## 핵심 프로세스 (풀 스톡테이크 기준)

### 1단계 — 현황 파악 (Inventory)
스킬 파일들을 나열하고 메타데이터(수정 시간 등)를 수집하여 대시보드 형태의 요약 표를 생성합니다.

### 2단계 — 품질 평가 (Quality Assessment)
에이전트(Opus 모델 권장)가 각 스킬을 읽고 다음 체크리스트를 기반으로 평가합니다:
* [ ] 다른 스킬과의 내용 중복 여부
* [ ] MEMORY.md / CLAUDE.md와의 중복 여부
* [ ] 기술적 참조의 최신성 (웹 검색으로 API, 도구 버전 등 확인)
* [ ] 사용 빈도 및 실용성

### 평가 판정 기준

| 판정 | 의미 |
|---------|---------|
| **Keep** | 유용하며 최신 상태 유지됨 |
| **Improve** | 유용하지만 특정 부분의 개선이 필요함 |
| **Update** | 참조하는 기술이 구식이므로 업데이트 필요 |
| **Retire** | 품질이 낮거나, 너무 오래되었거나, 비용 대비 가치가 없음 |
| **Merge into [X]** | 다른 스킬과 중복되므로 [X] 스킬로 통합 권장 |

## 평가 이유(Reason) 작성 규칙
* **단순 표현 지양**: "변경 없음" , "중복됨" 처럼 짧은 말 대신 구체적인 근거를 제시해야 합니다.
* **Retire의 경우**: 어떤 구체적인 결함이 있는지, 어떤 대안이 더 나은지 상세히 설명하십시오.
* **Merge의 경우**: 통합 대상 스킬의 이름을 명시하고, 어떤 내용을 어디로 합칠지 기재하십시오.
* **Improve의 경우**: 어느 부분을 어떻게 고쳐야 하는지 구체적인 가이드를 제공하십시오.

## 결과 요약 및 조치 (3~4단계)
1. **요약 표 제공**: 스킬 이름, 사용 빈도, 판정 결과, 근거를 표 형태로 보여줍니다.
2. **후속 조치 제안**: 은퇴(Retire)나 통합(Merge)이 필요한 경우, 사용자에게 최종 승인을 요청하며 조치를 수행합니다.
3. **최적화**: 내용이 너무 길거나 중복된 부분은 압축하고, 최신 기술 트렌드에 맞게 내용을 업데이트합니다.

**핵심**: 스킬 스톡테이크를 통해 에이전트의 효율성을 유지하고, 불필요한 비용 소모를 방지하십시오. 정기적으로 스킬의 품질을 관리하여 최상의 퍼포먼스를 유지하는 것이 중요합니다.
