# 프론트엔드 슬라이드 (Frontend Slides)

외부 의존성 없이 브라우저에서 즉시 실행되는 애니메이션 중심의 HTML 발표 자료를 생성합니다. 시각적 탐색 방법론을 통해 사용자에게 최적화된 미학을 제공합니다.

## 적용 시점

* 강연, 피칭, 워크숍 또는 내부 브리핑용 발표 자료를 만들 때
* `.ppt` 또는 `.pptx` 슬라이드를 웹 기반 HTML 발표 자료로 변환할 때
* 기존 HTML 슬라이드의 레이아웃, 모션, 타이포그래피를 개선하고 싶을 때
* 디자인 선호도가 명확하지 않은 사용자와 함께 스타일을 탐색하며 결과물을 도출할 때

## 타협할 수 없는 원칙

1. **제로 디펜던시 (Zero dependencies)**: 기본적으로 인라인 CSS와 JS를 포함하는 단일 HTML 파일로 구성합니다.
2. **뷰포트 최적화 (Viewport-fit)**: 각 슬라이드는 하나의 화면(Viewport)에 딱 맞아야 하며, 슬라이드 내부 스크롤은 금지됩니다.
3. **설명 대신 시각화 (Show, don't tell)**: 추상적인 질문 대신 실제 시각적 미리보기를 통해 스타일을 제안합니다.
4. **독창적인 디자인 (Distinctive design)**: 뻔한 보라색 그라데이션, 흰 배경에 Inter 폰트 같은 템플릿 느낌을 지양합니다.
5. **프로덕션 품질 (Production quality)**: 코드 주석은 명확해야 하며, 접근성(A11y), 반응형, 성능을 보장합니다.

*생성 전, 뷰포트 안전을 위한 CSS 기초, 밀도 제한, 스타일 프리셋 등이 담긴 `STYLE_PRESETS.md`를 반드시 읽어보십시오.*

## 워크플로우

### 1. 모드 감지 (Mode Detection)
* **신규 제작**: 주제, 노트 또는 전체 초안이 있는 경우
* **PPT 변환**: `.ppt` 또는 `.pptx` 파일이 있는 경우
* **강화 (Enhance)**: 기존 HTML 슬라이드가 있고 개선을 원하는 경우

### 2. 콘텐츠 파악
최소한의 핵심 정보만 확인합니다:
* **목적**: 피칭, 교육, 컨퍼런스 발표, 내부 업데이트 등
* **분량**: 짧음(5-10장), 중간(10-20장), 길음(20장 이상)
* **준비 상태**: 완성된 원고, 거친 메모, 주제만 있음 등

### 3. 스타일 탐색 (Style Discovery)
기본적으로 '시각적 탐색' 방식을 채택합니다.
1. 발표 자료가 풍겨야 할 분위기(인상적인, 에너제틱한, 집중력 있는, 영감을 주는 등)를 확인합니다.
2. `.ecc-design/slide-previews/` 경로에 **3가지 단일 슬라이드 미리보기 파일**을 생성합니다.
3. 각 미리보기는 타이포그래피, 색상, 애니메이션을 명확히 보여주어야 하며 100줄 내외의 코드로 작성합니다.
4. 어떤 스타일을 선택하거나 혼합할지 사용자에게 확인합니다.

### 4. 발표 자료 구축
최종 결과물은 `presentation.html` 또는 `[프로젝트명].html`로 저장합니다. 
* 의미론적(Semantic) 슬라이드 섹션 구성
* `STYLE_PRESETS.md`에 근거한 뷰포트 안전 CSS 적용
* 키보드, 휠, 터치 내비게이션 지원
* Intersection Observer를 활용한 등장 애니메이션

### 5. 뷰포트 맞춤 강제 적용
이 규칙은 타협할 수 없는 물리적 제약입니다:
* 각 `.slide`는 `height: 100vh; height: 100dvh; overflow: hidden;`을 가져야 합니다.
* 모든 폰트와 간격은 `clamp()`를 사용하여 화면 크기에 따라 가변적이어야 합니다.
* 내용이 넘치면 슬라이드를 분할하십시오. 텍스트 크기를 무리하게 줄이지 마십시오.

### 6. 검증 및 전달
* 1920x1080부터 모바일 규격까지 다양한 해상도에서 레이아웃 깨짐을 확인합니다.
* 키보드 내비게이션 작동 여부를 확인합니다.
* 전달 시 OS별 실행 명령(macOS: `open`, Windows: `start`)과 함께 커스터마이징 포인트를 안내합니다.

## PPT / PPTX 변환 가이드

1. 우선적으로 `python-pptx` 라이브러리를 사용하여 텍스트, 이미지, 노트를 추출합니다.
2. 슬라이드 순서와 발표자 노트를 보존합니다.
3. 추출 완료 후, 위 '스타일 탐색' 과정을 거쳐 디자인을 적용합니다.

## 기술 요구사항

### HTML / CSS
* 웹 폰트는 Google Fonts 또는 Fontshare를 권장합니다.
* 강렬한 타이포그래피 계층과 명확한 시각적 방향성을 우선합니다.
* 단순 일러스트보다는 추상적인 도형, 그라데이션, 노이즈, 기하학적 패턴을 활용하십시오.

### JavaScript
* 키보드 내비게이션 (좌우 화살표, Space)
* 터치/스와이프 및 마우스 휠 지원
* 진행률 표시(Progress indicator)
* `prefers-reduced-motion` 준수

### 슬라이드 밀도 제한 (Content Density)
| 슬라이드 유형 | 제한 사항 |
|------------|-------|
| 제목 | 메인 제목 + 부제목 + (선택) 태그라인 |
| 내용 | 제목 + 4~6개의 불렛포인트 또는 2개의 짧은 단락 |
| 그리드/카드 | 최대 6개 카드 |
| 코드 | 최대 8~10줄 |
| 인용구 | 인용 문구 + 출처 |

## 피해야 할 안티 패턴

* 개성 없는 뻔한 스타트업 그라데이션
* 스크롤이 생기는 긴 텍스트나 코드 블록
* 가독성을 해치는 너무 작은 폰트 크기
* 시끄럽고 산만한 과도한 애니메이션
* 특정 화면 크기에서 깨지는 고정 높이 레이아웃

## 전달 체크리스트

* [ ] 스타일이 독창적이고 의도적임
* [ ] 애니메이션이 의미 있게 작동함
