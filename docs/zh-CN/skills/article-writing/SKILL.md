# 기사 및 콘텐츠 작성 (Article Writing)

일반적인 AI 출력물이 아닌, 실제 사람이나 브랜드의 목소리가 느껴지는 장문의 콘텐츠를 작성합니다.

## 사용 시점

* 블로그 포스트, 에세이, 공지글, 가이드, 튜토리얼 또는 뉴스레터 초안을 작성할 때
* 메모, 녹취록 또는 조사 자료를 정제된 기사로 변환할 때
* 특정 창업자, 운영자 또는 브랜드의 기존 목소리 톤에 맞추어야 할 때
* 기존 장문 원고의 구조, 리듬 및 논거를 강화해야 할 때

## 핵심 규칙

1. **구체적인 것으로 시작**: 사례, 결과물, 일화, 데이터, 스크린샷 묘사 또는 코드 블록 등을 서두에 배치하십시오.
2. 예시를 먼저 보여준 뒤 설명하십시오.
3. 만연체보다는 짧고 명확한 문장을 선호하십시오.
4. 가능한 한 구체적이고 출처가 명확한 데이터를 사용하십시오.
5. 인물의 이력, 기업 지표 또는 고객 사례 등을 절대로 **조작하거나 지어내지 마십시오.**

## 목소리 톤(Voice) 파악 워크플로우

사용자가 특정 목소리 톤을 요청하는 경우, 다음 중 하나 이상의 자료를 수집하십시오:

* 이미 발표된 기사나 칼럼
* 뉴스레터
* X(트위터) / LinkedIn 포스트
* 문서 또는 내부 메모
* 간략한 스타일 가이드

자료를 통해 다음 요소들을 추출합니다:

* 문장의 길이와 리듬
* 어조의 성격 (격식 있는, 대화체, 날카로운 등)
* 선호하는 수사 기법 (괄호 사용, 목록화, 문장 끊기, 설의법 등)
* 유머, 주관적 견해, 반골 기질(Counter-intuitive frameworks)에 대한 허용 범위
* 포매팅 습관 (제목 스타일, 불렛 포인트, 코드 블록, 인용구 등)

참고할 만한 목소리 톤 자료가 없다면, 구체적이고 실용적이며 과장된 홍보 문구가 적은 '실무자 스타일'의 목소리를 기본으로 합니다.

## 금지 패턴 (Ban Patterns)

다음 내용이 발견되면 즉시 삭제하고 다시 작성하십시오:

* "오늘날 급변하는 환경 속에서"와 같은 진부한 도입부
* "게다가", "또한"과 같이 의미 없는 연결어(Filler words)
* "게임 체인저", "최첨단", "혁신적인"과 같이 과장된 형용사
* 근거 없는 모호한 주장
* 맥락 설명 없는 이력 나열이나 신뢰성 주장

## 집필 프로세스

1. 독자와 목적을 명확히 정의합니다.
2. 각 섹션의 목적이 뚜렷한 골조(Outline)를 잡습니다.
3. 각 섹션은 증거, 예시 또는 시나리오로 시작합니다.
4. 다음 문장이 존재해야 할 가치가 있는 지점에서만 내용을 확장하십시오.
5. 템플릿 느낌이 나거나 자화자찬하는 내용은 과감히 삭제하십시오.

## 구조 가이드

### 기술 가이드
* 독자가 얻을 수 있는 이득을 먼저 제시하십시오.
* 주요 섹션마다 코드나 터미널 실행 예시를 사용하십시오.
* 모호한 요약이 아닌 구체적인 실행 지침으로 마무리하십시오.

### 에세이 / 오피니언 리더용 기사
* 긴장감, 모순 또는 날카로운 관찰로 시작하십시오.
* 섹션당 하나의 논리적 흐름만 유지하십시오.
* 관점을 뒷받침하는 강력한 사례를 사용하십시오.

### 뉴스레터
* 첫 화면(Above the fold)에 강력한 내용을 배치하십시오.
* 단순한 일상 나열이 아닌, 업데이트 내용에 통찰(Insight)을 결합하십시오.
* 명확한 섹션 라벨과 훑어보기 좋은 구조를 사용하십시오.

## 품질 체크리스트

결과물 전달 전 확인 사항:

* 제공된 출처를 바탕으로 사실 관계를 확인했는가
* 의미 없는 미사여구와 기업용 상투어를 삭제했는가
* 목소리 톤이 제공된 예시와 일치하는가
* 각 섹션마다 새로운 정보가 추가되었는가
* 대상 플랫폼에 적합한 포맷을 적용했는가
