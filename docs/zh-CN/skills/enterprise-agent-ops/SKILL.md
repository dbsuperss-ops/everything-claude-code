# 기업급 에이전트 운영 (AgentOps)

단일 CLI 세션을 넘어, 클라우드 호스팅이나 지속적으로 실행되는 에이전트 시스템을 관리하고 통제할 때 이 스킬을 활용하십시오.

## 운영 영역

1. **런타임 생명주기**: 시작(Start), 일시 정지(Pause), 중지(Stop), 재시작(Restart) 관리
2. **관측 가능성**: 로그(Logs), 지표(Metrics), 추적(Tracing) 모니터링
3. **보안 제어**: 작업 범위(Scope) 제한, 권한 관리, 비상 정지(Kill-switch) 구현
4. **변경 관리**: 배포(Release), 롤백(Rollback), 감사(Audit) 프로세스

## 기본 통제 원칙 (Baseline Controls)

* **불변의 배포 산출물**: 변경이 불가능한 상태로 배포하여 일관성 유지
* **최소 권한 자격 증명**: 에이전트에 필요한 최소한의 권한만 부여
* **환경별 비밀 정보 주입**: 소스 코드가 아닌 환경 변수 등을 통한 보안 관리
* **강제 타임아웃 및 재시도 할당량**: 무한 루프 방지 및 리소스 최적화
* **고위험 작업 감사 로그**: 주요 동작에 대한 상세 이력 기록

## 추적 지표 (Metrics to Track)

* **성공률 (Success Rate)**: 에이전트의 임무 완수 비율
* **작업당 평균 재시도 횟수**: 효율성 측정 지표
* **복구 시간 (Time to Recover)**: 장애 발생 시 복구에 걸리는 시간
* **성공한 작업당 비용**: 경제성 평가 지표
* **장애 유형별 분포**: 반복되는 문제점 파악

## 장애 대응 패턴 (Incident Patterns)

장애가 급증할 경우 다음 단계를 따르십시오:

1. **신규 배포 중단**: 현재 상태 고정
2. **대표적인 트레이스 데이터 캡처**: 문제 원인 분석을 위한 데이터 수집
3. **장애 경로 격리**: 피해 확산 방지
4. **최소한의 보안 패치 적용**: 원인 해결
5. **회귀 테스트 및 보안 점검**: 패치 후 부작용 확인
6. **점진적 서비스 재개**: 단계별 복구

## 배포 연동

본 스킬은 다음 도구들과 함께 시너지를 낼 수 있습니다:

* PM2 워크플로우
* systemd 서비스 관리
* 컨테이너 오케스트레이터 (K8s 등)
* CI/CD 게이팅 시스템
