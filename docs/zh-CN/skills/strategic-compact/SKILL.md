# 전략적 압축 스킬 (Strategic Compact)

작업 흐름의 전략적 노드에서 수동으로 `/compact` 명령을 실행하여 컨텍스트 효율을 극대화하고 자동 압축으로 인한 정보 손실을 방지하십시오.

## 적용 시점

* 긴 세션을 진행하여 컨텍스트 제한(200K+ 토큰)에 도달할 때
* 다단계 작업(조사 → 계획 → 구현 → 테스트)을 처리할 때
* 동일한 세션 내에서 서로 다른 무관한 업무로 전환할 때
* 주요 마일스톤을 완료하고 새로운 단계로 넘어갈 때
* 응답 속도가 느려지거나 일관성이 떨어지는 느낌을 받을 때

## 왜 전략적 압축이 필요한가?

자동 압축은 시스템이 임의의 시점에 실행하므로 다음과 같은 문제가 발생할 수 있습니다:
* 작업 도중 중요한 맥락을 상실할 수 있음
* 논리적인 작업 경계(Job boundary)를 인식하지 못함
* 복잡한 다단계 작업을 방해할 수 있음

전략 거점에서 수동 압축을 하면:
* **조사 후, 실행 전**: 방대한 조사 데이터는 압축하고 정제된 구현 계획만 보존합니다.
* **마일스톤 완료 후**: 다음 단계를 위해 신선한 컨텍스트로 새로 시작합니다.
* **업무 전환 직전**: 이전 업무의 맥락이 새로운 업무를 오염시키지 않도록 정리합니다.

## 작동 방식

`suggest-compact.js` 스크립트가 도구 사용 전(`PreToolUse`) 실행되어 다음을 수행합니다:
1. **도구 호출 추적**: 세션 내의 도구 사용 횟수를 카운트합니다.
2. **임계값 감지**: 설정된 횟수(기본 50회) 도달 시 압축을 제안합니다.
3. **정기 알림**: 임계값 도달 이후 매 25회마다 다시 제안합니다.

## 압축 의사결정 가이드

| 단계 전환 시점 | 압축 여부 | 이유 |
| ------------------------ | ------ | -------------------------------------------------------------------- |
| 조사 → 계획              | Yes     | 조사는 방대하지만 계획은 정제된 결과물이므로 압축 가능 |
| 계획 → 구현              | Yes     | 계획은 파일에 저장되어 있으므로 코딩 공간 확보를 위해 압축 |
| 구현 → 테스트              | Maybe   | 최근 작성한 코드 맥락이 필요하면 유지, 아니면 압축 |
| 디버깅 → 다음 기능        | Yes     | 디버깅의 흔적은 다음 작업에 방해가 될 수 있음 |
| 구현 작업 및 파일 편집 중  | No     | 변수명, 파일 경로 등 세밀한 상태 정보 유지 필요 |
| 실패한 시도 이후           | Yes     | 잘못된 추론 과정을 지우고 새로운 시각으로 다시 시작 |

## 압축 후 보존되는 항목

* `CLAUDE.md` 지침 (항상 유지)
* TodoWrite 작업 리스트
* 기억 파일 (`~/.claude/memory/`)
* Git 상태 (커밋, 브랜치 등)
* 디스크에 저장된 실제 파일들

**핵심**: 압축은 '삭제'가 아닌 '정리'입니다. 중요한 정보는 파일이나 기억 파일에 먼저 저장한 후, `/compact`를 통해 에이전트의 사고 공간을 확보하십시오.
