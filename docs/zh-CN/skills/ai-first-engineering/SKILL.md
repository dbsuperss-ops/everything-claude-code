# AI 우선 (AI-First) 엔지니어링

AI 보조 코드 생성을 활용하는 팀의 프로세스, 리뷰 및 아키텍처를 설계할 때 이 스킬을 사용하십시오.

## 프로세스의 변화

1. 타이핑 속도보다 계획의 품질이 더 중요합니다.
2. 주관적인 자신감보다 평가(Eval) 커버리지가 더 중요합니다.
3. 리뷰의 초점이 문법에서 시스템 동작(System Behavior)으로 이동합니다.

## 아키텍처 요구 사항

AI 에이전트가 이해하기 쉬운 아키텍처를 우선시하십시오:

* 명확한 경계 (Explicit Boundaries)
* 안정적인 규약 (Stable Contracts)
* 타입화된 인터페이스 (Typed Interfaces)
* 결정론적인 테스트 (Deterministic Tests)

숨겨진 관례(Conventions)에 분산되어 있는 암시적 동작을 피하십시오.

## AI 우선 팀에서의 코드 리뷰

리뷰 시 다음 항목에 집중하십시오:

* 비헤이비어 회귀 (Behavioral Regression)
* 보안 관련 가정
* 데이터 무결성
* 오류 처리 (Fault Handling)
* 릴리스 안정성

자동화 도구가 이미 처리하고 있는 스타일 문제에 시간을 낭비하지 마십시오.

## 채용 및 역량 평가 지표

역량 있는 AI 우선 엔지니어는 다음과 같은 능력을 갖추고 있습니다:

* 모호한 작업을 명확하게 분해하는 능력
* 측정 가능한 인수 조건(Acceptance Criteria) 정의 능력
* 고가치 프롬프트 및 평가 항목 생성 능력
* 마감 압박 속에서도 리스크를 통제하는 능력

## 테스트 표준

AI가 생성한 코드에 대해서는 더 높은 수준의 테스트 표준을 적용하십시오:

* 관련 도메인에 대한 회귀 테스트 커버리지 요구
* 명확한 엣지 케이스 단언(Assertions)
* 인터페이스 경계에서의 통합 검사
