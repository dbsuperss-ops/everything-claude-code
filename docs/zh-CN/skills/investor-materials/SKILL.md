# 투자 유치 자료 (Investor Materials)

투자자들에게 전달할 자료는 일관성 있고 신뢰할 수 있으며, 논리적으로 방어 가능해야 합니다.

## 적용 시점

* 투자 유치를 위한 피치 덱(Pitch Deck)을 제작하거나 수정할 때
* 투자 메모(Investor Memo) 또는 한 페이지 요약문(One-pager)을 작성할 때
* 재무 모델, 마일스톤 계획 또는 자금 사용 계획서(Use of Funds)를 구축할 때
* 액셀러레이터 또는 인큐베이터 지원서 문항에 답할 때
* 여러 개의 투자 관련 문서들을 단일 정보 소스(Single Source of Truth)를 기준으로 통합할 때

## 황금률 (Golden Rule)

**모든 투자 자료는 서로 내용이 일치해야 합니다.**

작성 시작 전에 반드시 단일 정보 소스를 정의하거나 확인하십시오:
* 성장 지표 (Traction metrics)
* 가격 정책 및 매출 가이드라인
* 펀딩 규모 및 조달 수단
* 자금의 용도 (Use of funds)
* 팀 구성 및 직함
* 마일스톤 및 타임라인

만약 문서 간에 숫자가 충돌한다면, 즉시 작성을 멈추고 이를 해결해야 합니다.

## 핵심 워크플로우

1. 공식 팩트(Facts) 목록 확인
2. 누락된 가설(Assumptions) 식별
3. 필요한 문서 유형 선택
4. 명확한 논리에 기반한 초안 작성
5. 정보 소스와 각 숫자를 교차 검증

## 주요 문서 가이드라인

### 피치 덱 (Pitch Deck)
권장 구성:
1. 회사 개요 + 핵심 가치
2. 문제 정의 (Problem)
3. 해결책 (Solution)
4. 제품 / 데모
5. 시장 규모 (Market)
6. 비즈니스 모델
7. 성장 지표 (Traction)
8. 팀 (Team)
9. 경쟁 / 차별화 포인트
10. 필요한 투자 규모 (The Ask)
11. 자금 용도 / 마일스톤
12. 부록 (Appendix)

*웹 기반 슬라이드가 필요한 경우 `frontend-slides` 스킬과 병행하십시오.*

### 한 페이지 요약 (One-pager) / 메모
* 회사가 무엇을 하는지 한 문장으로 명확히 설명하십시오.
* '왜 지금인가(Why now)'를 보여주십시오.
* 성장 수치와 증거(Proof points)를 초반에 배치하십시오.
* 투자 요청 금액을 정확히 기재하십시오.
* 주장이 쉽게 검증 가능하도록 작성하십시오.

### 재무 모델 (Financial Model)
포함 사항:
* 명확한 가설(Assumptions) 명시
* 필요 시 시나리오 분석(Pessimistic / Base / Optimistic) 포함
* 매출 발생 논리 (Revenue logic) 기술
* 마일스톤과 연계된 지출 계획
* 주요 가설에 따른 민감도 분석(Sensitivity analysis)

### 액셀러레이터 지원서
* 질문의 의도에 정확히 답변하십시오.
* 성장 지표, 인사이트, 팀의 강점을 우선순위에 두십시오.
* 과장된 표현은 피하십시오.
* 피치 덱이나 재무 모델의 지표와 동기화하십시오.

## 피해야 할 위험 신호 (Red Flags)

* 검증 불가능한 주장
* 가설 없는 막연한 시장 규모 추정
* 문서마다 다른 팀 역할이나 직함
* 매출 계산 근거 불명확
* 취약한 가설을 확신이 있는 것처럼 포장하는 행위

## 최종 품질 검토
전달 전 다음 확인 사항을 점검하십시오:
* [ ] 모든 숫자가 정보 소스와 일치하는가
* [ ] 자금 용도 및 매출 계산 로직이 정확한가
* [ ] 가설이 숨겨져 있지 않고 명시되어 있는가
* [ ] 수식어나 과장 없이 담백하게 스토리가 전달되는가
* [ ] 최종 결과물이 파트너 미팅에서 논리적으로 방어 가능한가
