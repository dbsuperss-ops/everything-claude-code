# 에이전틱 엔지니어링 (Agentic Engineering)

AI 에이전트가 구현 작업의 대부분을 수행하고, 인간은 품질 및 리스크 제어를 담당하는 엔지니어링 워크플로우에서 이 스킬을 사용하십시오.

## 운영 원칙

1. 실행 전 완료 기준을 정의하십시오.
2. 작업을 에이전트가 처리 가능한 단위로 분해하십시오.
3. 작업 복잡도에 따라 적절한 모델 계층으로 라우팅하십시오.
4. 평가(Eval) 및 회귀 검사를 통해 성과를 측정하십시오.

## 평가 우선 루프 (Eval-First Loop)

1. 역량 평가 및 회귀 평가 항목을 정의하십시오.
2. 베이스라인을 실행하고 실패 특성을 캡처하십시오.
3. 구현을 실행하십시오.
4. 평가를 다시 실행하고 차이점(Diff)을 비교하십시오.

## 작업 분해 (Task Decomposition)

'15분 단위 규칙'을 적용하십시오:

* 각 단위는 독립적으로 검증 가능해야 합니다.
* 각 단위는 하나의 주요 리스크를 가져야 합니다.
* 각 단위는 명확한 완료 조건을 제시해야 합니다.

## 모델 라우팅 (Model Routing)

* **Haiku**: 분류, 보일러플레이트 변환, 좁은 범위의 편집
* **Sonnet**: 구현 및 리팩토링
* **Opus**: 아키텍처, 근본 원인 분석, 다중 파일 불변성 검사

## 세션 전략

* 밀접하게 결합된 작업 단위의 경우 동일한 세션을 유지하십시오.
* 주요 단계가 전환된 후에는 새로운 세션을 시작하십시오.
* 활발하게 디버깅 중일 때가 아니라, 마일스톤이 완료된 시점에 컨텍스트 압축을 수행하십시오.

## AI 생성 코드의 리뷰 중점 사항

다음을 우선적으로 리뷰하십시오:

* 불변성(Invariants) 및 엣지 케이스
* 에러 경계(Error boundaries)
* 보안 및 인증 관련 가정
* 숨겨진 결합도 및 배포 리스크

자동 포매팅/린팅 도구가 코드 스타일을 강제하고 있다면, 단순한 스타일 차이에 대한 리뷰로 시간을 낭비하지 마십시오.

## 비용 규율 (Cost Discipline)

작업별로 다음 항목을 추적하십시오:

* 사용된 모델
* 토큰 추정치
* 재시도 횟수
* 실제 소요 시간
* 성공/실패 여부

낮은 계층의 모델이 실패하고 명확한 추론 격차가 있을 때만 상위 모델로 업그레이드하십시오.
