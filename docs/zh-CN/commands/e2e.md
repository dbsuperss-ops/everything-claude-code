# E2E 명령어

이 명령어는 **e2e-runner** 에이전트를 호출하여 Playwright를 이용한 E2E 테스트의 생성, 유지 관리 및 실행을 수행합니다.

## 주요 기능

1. **테스트 여정 생성** - 사용자 흐름에 맞춘 Playwright 테스트 코드를 생성합니다.
2. **E2E 테스트 실행** - 다양한 브라우저 환경에서 테스트를 실행합니다.
3. **결과물(Artifacts) 캡처** - 테스트 실패 시 스크린샷, 비디오, 트레이스(Trace)를 자동으로 기록합니다.
4. **결과 보고** - HTML 리포트 및 CI용 JUnit XML을 생성합니다.
5. **불안정한 테스트(Flaky tests) 감지** - 간헐적으로 실패하는 테스트를 식별하여 격리하거나 수정 방안을 제시합니다.

## 적용 시점

`/e2e` 명령어를 사용하는 경우:

* 핵심 사용자 여정(로그인, 거래, 결제 등)을 테스트할 때
* 여러 단계로 구성된 프로세스가 처음부터 끝까지 정상 작동하는지 확인할 때
* UI 상호작용 및 내비게이션을 검증할 때
* 프론트엔드와 백엔드 간의 통합을 실질적으로 확인할 때
* 프로덕션 배포 전 최종 점검을 수행할 때

## 작동 원리

e2e-runner 에이전트는 다음 과정을 수행합니다:

1. **사용자 흐름 분석**: 테스트 시나리오를 식별합니다.
2. **POM(Page Object Model) 적용**: 유지보수가 용이한 구조로 Playwright 테스트 코드를 생성합니다.
3. **멀티 브라우저 실행**: Chromium, Firefox, WebKit(Safari) 환경에서 테스트를 구동합니다.
4. **디버깅 정보 수집**: 실패 지점의 스크린샷, 비디오, 단계별 실행 기록(Trace)을 확보합니다.
5. **리포트 생성**: 시각적인 HTML 보고서를 제공합니다.

## 실행 예시

````text
사용자: /e2e 시장 검색 및 상세 보기 흐름 테스트해줘

에이전트 (e2e-runner):
# E2E 테스트 생성: 시장 검색 및 상세 보기

## 식별된 테스트 시나리오
**사용자 여정**: 시장 검색 → 결과 확인 → 특정 시장 클릭 → 상세 정보 확인

## 생성된 테스트 코드 (예시)
```typescript
// tests/e2e/markets/search-and-view.spec.ts
import { test, expect } from '@playwright/test'

test('사용자가 시장을 검색하고 상세 정보를 확인할 수 있음', async ({ page }) => {
  // 1. 시장 페이지 이동 및 제목 확인
  // 2. 검색 수행 (예: '선거')
  // 3. 검색 결과 노출 확인 (스크린샷 저장)
  // 4. 첫 번째 결과 클릭
  // 5. 상세 페이지 로드 및 차트 렌더링 확인
})
```
````

## 결과물 확인 방법

```bash
# 브라우저에서 HTML 리포트 보기
npx playwright show-report

# 특정 트레이스 파일 분석
npx playwright show-trace artifacts/trace-abc123.zip

# 스크린샷 확인
open artifacts/search-results.png
```

## 베스트 프랙티스

* **권장(Do)**: 
  * 유지보수를 위해 **Page Object Model**을 사용하십시오.
  * 선택자(Selector)로 `data-testid` 속성을 활용하십시오.
  * 임의의 타임아웃 대신 API 응답 대기(`waitForResponse`)를 사용하십시오.
  * 핵심 사용자 여정을 중심으로 테스트를 구성하십시오.
* **지양(Don't)**:
  * 부서지기 쉬운 CSS 클래스 기반 선택자를 쓰지 마십시오.
  * 내부 구현 상세를 테스트하지 마십시오.
  * 프로덕션 환경에서 직접 거래 테스트를 수행하지 마십시오.
  * 모든 에지 케이스를 E2E로 테스트하려 하지 마십시오 (단위 테스트 활용 권장).

**중요 (금융/거래 앱)**: 실제 자금이 연동된 테스트는 반드시 **테스트넷/스테이징 환경**에서만 실행하십시오. 프로덕션 환경에서의 거래 테스트는 엄격히 금지됩니다.

## 관련 도구 및 명령어

* `/plan`: 테스트할 핵심 여정을 식별하는 데 사용
* `/tdd`: 더 빠르고 세밀한 단위 테스트(Unit Test)를 위해 사용
* `/code-review`: 작성된 테스트 코드의 품질을 검증할 때 사용
* `e2e-runner` 에이전트(`~/.claude/agents/e2e-runner.md`)가 실제 로직을 처리합니다.
