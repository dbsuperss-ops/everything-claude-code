# 검증 (Verify) 명령어

코드의 무결성을 보장하기 위해 빌드부터 보안 스캔까지 일련의 자동화된 검사를 실행합니다.

## 검증 순서 및 항목

1. **빌드 검사 (Build Check)**
   * 프로젝트의 빌드 명령어를 실행합니다.
   * 실패 시 즉시 중단하고 원인을 보고합니다.

2. **타입 검사 (Type Check)**
   * TypeScript 또는 각 언어의 타입 체커를 가동합니다.
   * 파일명과 라인 넘버를 포함한 에러 리스트를 작성합니다.

3. **린트 검사 (Lint Check)**
   * 정적 코드 분석 도구(Linter)를 실행합니다.
   * 경고(Warning) 및 에러(Error) 내역을 보고합니다.

4. **테스트 수트 (Test Suite)**
   * 모든 테스트를 실행합니다.
   * 통과/실패 수량 및 커버리지 비율을 명시합니다.

5. **로그 감사 (Log Audit)**
   * 소스 코드 내 불필요한 `console.log` 사용 여부를 탐색합니다.
   * 발견된 위치를 보고합니다.

6. **Git 상태 (Git Status)**
   * 아직 커밋되지 않은 변경 사항과 최근 수정된 파일을 표시합니다.

---

## 검증 리포트 예시

성공 또는 실패 여부를 담은 간결한 보고서를 생성합니다:

```text
검증 결과 요약: [통과/실패]

빌드:        [OK / 실패]
타입 체크:   [OK / X개 에러]
린트:        [OK / X개 이슈]
테스트:      [X개 통과 / Y개 실행, Z% 커버리지]
비밀 정보:   [OK / X개 보안 노출]
로그 감사:   [OK / X개 console.log 발견]

풀 리퀘스트(PR) 준비 완료: [예 / 아니오]
```

발견된 크리티컬 이슈가 있다면 즉시 수정 권고 사항을 함께 제공합니다.

---

## 매개변수 (Arguments)

* `quick`: 빌드 및 타입 체크만 수행 (가장 빠른 확인용)
* `full`: 모든 검사 항목 수행 (기본값)
* `pre-commit`: 커밋 전 필수 권장 항목 검사
* `pre-pr`: 전체 검사에 보안 스캔이 추가된 PR 전용 상세 검사

**핵심**: Verify 명령어는 코드가 '작동한다'는 확신을 주기 위한 최종 방어선입니다. PR을 올리거나 배포를 준비할 때 반드시 실행할 것을 권장합니다.
