# 품질 관문 (Quality-Gate) 명령어

지정된 경로 또는 전체 프로젝트를 대상으로 코드 품질 검사를 수행합니다.

## 사용법

`/quality-gate [경로|.] [--fix] [--strict]`

* **대상 경로**: 기본값은 현재 디렉토리(`.`)입니다.
* `--fix`: 설정된 항목에 대해 자동 포맷팅이나 사소한 수정을 허용합니다.
* `--strict`: 엄격 모드 (경고 발생 시 실패로 간주) 여부를 선택합니다.

## 파이프라인 단계

1. 대상 파일의 **언어 및 도구**를 자동으로 감지합니다.
2. **포맷팅(Formatting)** 상태를 점검합니다.
3. 사용 가능한 경우 **코드 린트(Linting) 및 타입 체크(Type Checking)**를 수행합니다.
4. 발견된 문제점과 그에 대한 **간결한 수정 리스트**를 생성합니다.

## 매개변수 (Arguments)

* `[경로|.]`: (선택 사항) 검사를 수행할 대상 경로
* `--fix`: (선택 사항) 자동 수정 허용 여부
* `--strict`: (선택 사항) 엄격 검사 여부

**핵심**: Quality-Gate 명령어는 커밋 전 최종 점검이나 특정 모듈의 완성도를 보장하기 위한 '필터' 역할을 수행합니다. 수동으로 실행하여 코드의 무결성을 확보하십시오.
