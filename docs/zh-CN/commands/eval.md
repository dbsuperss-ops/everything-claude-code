# 평가 (Eval) 명령어

평가 기반 개발 워크플로우를 관리하고 실행합니다.

## 사용법

`/eval [define|check|report|list] [기능명]`

## 평가 정의 (Define)

`/eval define 기능명`

새로운 평가 정의를 생성합니다:

1. 템플릿을 사용하여 `.claude/evals/기능명.md` 파일을 생성합니다:
   ```markdown
   ## EVAL: 기능명
   생성일: $(date)

   ### 역량 평가 (Capability Eval)
   - [ ] [역량 1에 대한 상세 설명]
   - [ ] [역량 2에 대한 상세 설명]

   ### 회귀 평가 (Regression Eval)
   - [ ] [기존 동작 1이 여전히 유효함]
   - [ ] [기존 동작 2가 여전히 유효함]

   ### 성공 기준 (Success Criteria)
   - 역량 평가 pass@3 > 90%
   - 회귀 평가 pass^3 = 100%
   ```
2. 사용자에게 구체적인 기준 입력을 요청합니다.

## 평가 점검 (Check)

`/eval check 기능명`

기능에 대한 평가를 실행합니다:

1. `.claude/evals/기능명.md`에서 평가 정의를 읽어옵니다.
2. **역량 평가** 수행:
   * 각 기준을 검증하고 통과/실패 여부를 기록합니다.
   * `.claude/evals/기능명.log`에 시도 이력을 남깁니다.
3. **회귀 평가** 수행:
   * 관련 테스트를 실행하고 기준점(Baseline)과 비교합니다.
4. 현재 상태를 보고합니다:
   ```
   [평가 점검 결과]: 기능명
   ========================
   역량 검증: X/Y 통과
   회귀 검증: X/Y 통과
   상태: [진행 중 / 준비 완료]
   ```

## 평가 보고 (Report)

`/eval report 기능명`

종합적인 평가 보고서를 생성합니다:

```
[평가 보고서]: 기능명
=========================
생성일: $(date)

역량 평가 상세
----------------
[항목 1]: 성공 (pass@1)
[항목 2]: 성공 (pass@2) - 재시도 필요함
[항목 3]: 실패 - 비고란 참조

회귀 평가 상세
----------------
[테스트 1]: 성공
[테스트 2]: 성공
[테스트 3]: 성공

지표 (Metrics)
-------
역량 pass@1: 67%
역량 pass@3: 100%
회귀 pass^3: 100%

비고 (Notes)
-----
[이슈, 특이사항, 관찰된 엣지 케이스 등]

최종 권고
--------------
[배포 가능 / 보완 필요 / 차단됨]
```

## 평가 목록 (List)

`/eval list`

정의된 모든 평가 목록을 보여줍니다:

```
[평가 정의 목록]
================
기능-인증      [3/5 통과] 진행 중
기능-검색      [5/5 통과] 준비 완료
기능-내보내기  [0/4 통과] 시작 전
```

## 매개변수 (Arguments)

* `define <이름>` - 새로운 평가 정의 생성
* `check <이름>` - 평가 실행 및 점검
* `report <이름>` - 전체 보고서 작성
* `list` - 저장된 모든 평가 보기
* `clean` - 오래된 평가 로그 정리 (최근 10회 실행 이력만 보존)

**핵심**: 평가(Eval)는 단순한 테스트를 넘어, 기능이 의도한 역량을 실제로 갖추었는지 객관적으로 증명하는 도구입니다. 배포 전 최종 관문으로 활용하십시오.
