# 코드맵 업데이트 (Update-Codemaps) 명령어

코드베이스 전체를 스캔하여 시스템의 구조, 데이터 흐름, 의존성 관계를 일목요연하게 정리한 아키텍처 문서를 생성합니다.

## 1단계: 프로젝트 구조 스캔

1. **프로젝트 유형** 식별 (Monorepo, Single App, Library, Microservices 등)
2. 모든 **소스 코드 디렉토리** (`src/`, `lib/`, `app/`, `packages/` 등) 확인
3. **엔트리 포인트(Entry Points)** 매핑 (`main.ts`, `index.ts`, `app.py`, `main.go` 등)

## 2단계: 코드맵 생성 및 갱신

`docs/CODEMAPS/` (또는 `.reports/codemaps/`) 경로에 다음 파일들을 생성하거나 업데이트합니다:

| 파일명 | 주요 내용 |
|------|----------|
| `architecture.md` | 하위 시스템 구성도, 서비스 경계, 전체 데이터 흐름 |
| `backend.md` | API 라우트, 미들웨어 체인, 서비스 ↔ 저장소(Repo) 매핑 |
| `frontend.md` | 페이지 트리, 컴포넌트 계층 구조, 상태 관리 흐름 |
| `data.md` | 데이터베이스 테이블 구조, 관계도, 마이그레이션 이력 |
| `dependencies.md` | 외부 서비스 연동, 제3자 라이브러리, 공유 라이브러리 |

### 코드맵 작성 규칙 (AI 최적화)
AI가 토큰을 최소화하면서 정보를 효과적으로 소비할 수 있도록 **간결한 스타일**을 유지합니다:

```markdown
# 백엔드 아키텍처 요약

## 주요 라우트 (Routes)
POST /api/users → UserController.create → UserService.create → UserRepo.insert
GET  /api/users/:id → UserController.get → UserService.findById

## 핵심 소스 파일
src/services/user.ts (비즈니스 로직 중심)
src/repos/user.ts (DB 접근 레이어)

## 외부 의존성
- PostgreSQL (메인 스토리지)
- Redis (세션 캐시, 속도 제한)
- Stripe (결제 처리 연동)
```

## 3단계: 변경 사항 감지 및 승인

1. 기존 코드맵이 있는 경우 **차이점(Diff)**을 계산합니다.
2. 변경 사항이 **30%를 초과**하면 사용자에게 승인을 요청한 후 덮어씁니다.
3. 변경 사항이 **30% 이하**인 경우 즉시 업데이트를 진행합니다.

## 4단계: 메타데이터 및 분석 보고서 저장

* 각 문서 상단에 생성 일시, 스캔된 파일 수, 추정 토큰 수 등을 기록합니다.
* `.reports/codemap-diff.txt`에 지난 스캔 이후의 변경 요약(추가/삭제된 파일, 신규 의존성, 아키텍처 변화 등)을 저장합니다.

## 권장 사항

* 세부 구현보다는 **상위 레벨의 구조**에 집중하십시오.
* 코드 블록보다는 **파일 경로와 함수 시그니처**를 우선적으로 활용하십시오.
* 각 문서는 AI의 효율적인 컨텍스트 로드를 위해 **1,000 토큰 이내**로 유지하십시오.
* 주요 기능을 추가하거나 대규모 리팩토링을 마친 후 실행하는 것을 권장합니다.

**핵심**: 코드맵 업데이트 명령어는 에이전트가 프로젝트의 '지도'를 최신 상태로 유지하게 함으로써, 작업 시 발생할 수 있는 구조적 오해를 방지하고 탐색 효율을 극대화합니다.
