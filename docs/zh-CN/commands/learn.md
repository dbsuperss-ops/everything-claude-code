# /learn - 재사용 가능 패턴 추출

현재 세션을 분석하여 나중에 다른 작업에서도 활용할 수 있는 가치 있는 패턴을 추출하고 스킬로 저장합니다.

## 실행 시점

세션 진행 중 또는 종료 시, **단순하지 않은(Non-trivial) 문제를 해결했거나 새로운 방식**을 발견했을 때 즉시 `/learn`을 실행하십시오.

## 추출 대상 (무엇을 배우는가?)

1. **에러 해결 패턴**
   * 어떤 에러가 발생했는가?
   * 근본 원인(Root Cause)은 무엇인가?
   * 어떤 방법이 이를 해결했는가?
   * 유사한 에러 해결에 재사용 가능한가?

2. **디버깅 기법**
   * 일반적이지 않은 분석 단계
   * 효과적인 도구 조합
   * 진단 패턴 및 워크플로우

3. **우회 방법 (Workarounds)**
   * 특정 라이브러리의 특이점 해결
   * API 제약 사항 극복 방식
   * 특정 버전 전용 수정 사항

4. **프로젝트 특화 패턴**
   * 해당 코드베이스만의 고유한 컨벤션
   * 아키텍처 결정 사항 및 근거
   * 통합(Integration) 패턴

## 출력 파일 형식

`~/.claude/skills/learned/[패턴명].md` 경로에 스킬 파일을 생성합니다:

```markdown
# [설명적인 패턴 이름]

**추출일:** [날짜]
**문맥:** [이 패턴이 적용되는 상황 요약]

## 문제 상황
[해결하고자 하는 구체적인 문제 설명]

## 해결 방법
[기술적 해결책 또는 우회 기법]

## 예시
[해당하는 경우 코드 예시 포함]

## 적용 시점
[이 스킬을 활성화해야 하는 구체적인 트리거 조건]
```

## 처리 절차

1. 현재 세션 전체를 훑으며 추출 가능한 패턴을 탐색합니다.
2. 가장 가치 있고 재사용 가능한 인사이트를 선별합니다.
3. 스킬 파일 초안을 작성합니다.
4. 저장하기 전 사용자에게 내용을 확인받습니다.
5. `~/.claude/skills/learned/` 경로에 최종 저장합니다.

**주의 사항**:
* 단순한 오타 수정이나 기초적인 문법 에러는 저장하지 마십시오.
* 일회성 외부 서비스 장애와 같은 일시적인 이슈는 지양하십시오.
* 미래의 작업 시간을 단축할 수 있는 '패턴'에 집중하십시오.
* 하나의 스킬 파일에는 하나의 명확한 패턴만 담으십시오.

**핵심**: Learn 명령어는 에이전트가 단기 기억(세션)을 장기 기억(스킬)으로 변환하여 시간이 지날수록 더 똑똑해지게 만드는 핵심 도구입니다.
