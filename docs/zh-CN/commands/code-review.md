# 코드 리뷰 (Code Review)

커밋되지 않은 코드 변경 사항에 대해 종합적인 보안 및 품질 검토를 수행합니다.

## 검토 항목

### 1. 보안 이슈 (심각 - CRITICAL)
* 하드코딩된 자격 증명, API 키, 토큰 노출 여부
* SQL 인젝션 취약점
* XSS(교차 사이트 스크립팅) 취약점
* 입력값 검증(Validation) 누락
* 안전하지 않은 의존성(Dependencies) 사용
* 경로 탐색(Path Traversal) 공격 위험

### 2. 코드 품질 (높음 - HIGH)
* 함수 길이가 50라인을 초과하는 경우
* 파일 전체 길이가 800라인을 초과하는 경우
* 중첩 깊이(Nesting)가 4단계를 초과하는 경우
* 예외/에러 처리(Error Handling) 누락
* 불필요한 `console.log` 문 포함
* `TODO` / `FIXME` 주석 방치
* 공개 API에 JSDoc(문서화) 누락

### 3. 베스트 프랙티스 (중간 - MEDIUM)
* 가변성(Mutation) 패턴 사용 (불변성 패턴 권장)
* 코드나 주석 내 불필요한 이모지 사용
* 신규 코드에 대한 테스트 코드 누락
* 접근성(A11y) 가이드라인 위반

## 워크플로우

1. `git diff --name-only HEAD`를 통해 변경된 파일 목록을 확보합니다.
2. 각 파일에 대해 위 검토 항목을 기반으로 분석합니다.
3. 결과 보고서를 생성합니다:
   * **심각도**: 심각, 높음, 중간, 낮음
   * **위치**: 파일 경로 및 해당 라인 번호
   * **문제 설명**: 구체적인 위반 내용
   * **권장 조치**: 수정 방법 제안
4. **심각(CRITICAL)** 또는 **높음(HIGH)** 등급의 이슈가 발견되면 수정을 마칠 때까지 커밋을 중단하도록 권고합니다.

**핵심**: 보안 취약점이나 품질 저하가 포함된 코드는 절대로 승인되어서는 안 됩니다. 시스템의 안정성을 위해 리뷰 결과를 엄격히 준수하십시오.
