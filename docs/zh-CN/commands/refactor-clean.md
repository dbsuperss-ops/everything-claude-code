# 리팩토링 및 클린업 (Refactor-Clean)

테스트 기반의 안전한 절차를 통해 불필요한 코드를 식별하고 제거하여 코드베이스의 건강성을 유지합니다.

## 1단계: 데드 코드(Dead Code) 감지

프로젝트 유형에 적합한 분석 도구를 실행합니다:

| 도구 | 주요 점검 항목 | 실행 명령어 |
|------|--------------|---------|
| **knip** | 미사용 export, 파일, 의존성 | `npx knip` |
| **depcheck** | 미사용 npm 패키지 | `npx depcheck` |
| **ts-prune** | 미사용 TypeScript export | `npx ts-prune` |
| **vulture** | 미사용 Python 코드 | `vulture src/` |
| **deadcode** | 미사용 Go 코드 | `deadcode ./...` |

도구가 없는 경우, Grep을 사용하여 가져오기(import) 기록이 전혀 없는 export 구문을 직접 찾습니다.

## 2단계: 발견 사항 분류

발견된 코드를 안전성 등급에 따라 분류합니다:

* **🟢 안전 (Safe)**: 유틸리티 함수, 테스트 헬퍼, 내부 전용 함수 등 (즉각 삭제 가능)
* **🟡 주의 (Caution)**: 컴포넌트, API 라우트, 미들웨어 (동적 로딩 여부 확인 필수)
* **🔴 위험 (Danger)**: 설정 파일, 입구(Entry) 포인트, 기초 타입 정의 (정밀 조사 필요)

## 3단계: 안전한 단계별 삭제 루프

각 **안전** 등급 항목에 대해 다음을 반복합니다:

1. **전체 테스트 실행**: 기준점(Base)이 모두 통과하는지 확인합니다.
2. **코드 삭제**: 편집 도구를 사용하여 해당 항목을 정확히 제거합니다.
3. **테스트 재실행**: 삭제 후 기능이 망가지지 않았는지 검증합니다.
4. **실패 시 복구**: 테스트가 실패하면 즉시 `git checkout -- <파일>`로 복구하고 해당 건은 건너뜁니다.
5. **성공 시 유지**: 다음 항목으로 진행합니다.

## 4단계: '주의' 항목 처리 가이드

**주의** 등급의 코드를 삭제하기 전 체크리스트:
* 동적 임포트(`import()`, `require()`) 사용 여부 검색
* 설정 파일 등에서 문자열 기반 참조(컴포넌트명 등) 여부 확인
* 외부 공개용 패키지(Public API)의 일부인지 검토

## 5단계: 중복 코드 병합

삭제 작업 후 다음을 추가로 점검합니다:
* 유사도 80% 이상의 중복 함수 병합
* 파편화된 타입 정의(Type definitions) 통합
* 단순 래핑만 하는 무의미한 함수 인라인(Inline) 처리

## 6단계: 최종 결과 요약

작업이 완료되면 다음과 같이 보고합니다:

```text
데드 코드 클린업 완료 보고
──────────────────────────────
삭제됨:   12개 함수 (미사용)
          3개 파일 (미사용)
          5개 패키지 (미사용 의존성)
건너뜀:   2개 항목 (테스트 실패)
감소량:   약 450줄 제거
──────────────────────────────
최종 상태: 모든 테스트 통과 ✅
```

**핵심 규칙**:
* **테스트 없이 삭제하지 마십시오.** (검증이 필수입니다)
* **한 번에 하나씩만 처리하십시오.** (원자적 변경이 복구에 유리합니다)
* **불확실하면 건너뛰십시오.** (운영 환경을 망치는 것보다 사용하지 않는 코드를 두는 편이 낫습니다)
* **클린업과 리팩토링을 동시에 하지 마십시오.** (정리와 재작성은 분리해야 합니다)
