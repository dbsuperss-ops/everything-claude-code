# 에이전트 오케스트레이션 (Agent Orchestration)

## 사용 가능한 에이전트

`~/.claude/agents/` 디렉토리에 위치합니다:

| 에이전트 | 용도 | 사용 시기 |
|-------|---------|-------------|
| planner | 구현 계획 수립 | 복잡한 기능 개발, 리팩토링 |
| architect | 시스템 설계 | 아키텍처 결정 |
| tdd-guide | 테스트 주도 개발 | 신규 기능 개발, 버그 수정 |
| code-reviewer | 코드 리뷰 | 코드 작성 완료 후 |
| security-reviewer | 보안 분석 | 커밋 전 |
| build-error-resolver | 빌드 에러 수정 | 빌드 실패 시 |
| e2e-runner | 엔드투엔드 테스트 | 주요 사용자 워크플로우 테스트 |
| refactor-cleaner | 데드 코드 정리 | 코드 유지보수 |
| doc-updater | 문서화 | 문서 업데이트 시 |

## 즉시 에이전트 활용 가이드

사용자 요청이 없더라도 다음 상황에서는 에이전트를 활용하십시오:

1. 복잡한 기능 구현 요청 - **planner** 에이전트 사용
2. 방금 작성/수정한 코드 - **code-reviewer** 에이전트 사용
3. 버그 수정 또는 신규 기능 - **tdd-guide** 에이전트 사용
4. 아키텍처 관련 결정 - **architect** 에이전트 사용

## 병렬 작업 실행

서로 독립적인 작업의 경우, **항상** 병렬 실행을 활용하십시오:

```markdown
# 좋은 예: 병렬 실행
다음과 같이 3개의 에이전트를 동시에 실행합니다:
1. 에이전트 1: 인증 모듈 보안 분석
2. 에이전트 2: 캐시 시스템 성능 검토
3. 에이전트 3: 유틸리티 클래스 타입 체크

# 나쁜 예: 불필요한 순차 실행
에이전트 1 실행 완료 후 에이전트 2 실행, 그 다음 에이전트 3 실행
```

## 다각도 분석

복잡한 문제의 경우, 역할을 나눈 서브 에이전트들을 활용하십시오:

* 팩트 체크 전문가 (Fact Reviewer)
* 시니어 엔지니어 (Senior Engineer)
* 보안 전문가 (Security Expert)
* 일관성 검토 전문가 (Consistency Reviewer)
* 중복 코드 체크 도구 (Redundancy Checker)
